# SPEC: Логика уведомлений

## Цель
Отправлять Telegram-уведомления, когда топливо на АЗС стало доступно после недоступности.

## Отслеживаемые поля
- `a92`
- `a95`
- `a95_ultra`
- `a100`
- `diesel`
- `diesel_ultra`
- `lpg`

## Допустимые статусы
- `FUEL_STATUS_IN_STOCK`
- `FUEL_STATUS_LIMITED`
- `FUEL_STATUS_OUT_OF_STOCK`
- `FUEL_STATUS_UNAVAILABLE`

## Правило "появилось топливо"
Событие фиксируется, если статус перешел:
- из любого статуса, кроме `FUEL_STATUS_IN_STOCK`
- в `FUEL_STATUS_IN_STOCK`
- По умолчанию это задается через `NOTIFY_TARGET_STATUSES=["FUEL_STATUS_IN_STOCK"]`.

## Алгоритм цикла
1. Запросить текущий список АЗС.
2. Прочитать предыдущее состояние.
3. Для каждой АЗС и каждого вида топлива сравнить старый и новый статус.
4. Сформировать события по правилу перехода.
5. Отправить уведомления в Telegram.
6. Сохранить новое состояние.

## Дедупликация уведомлений
- Для ключа `station_uuid + fuel_type` вводится cooldown (рекомендовано 30 минут).
- Если событие повторилось в cooldown, уведомление не отправляется.
- Первый запуск: либо без рассылки (safe mode), либо с рассылкой всех текущих доступных позиций (инициализационный режим).

## Обработка ошибок
- Ошибка API: логировать и повторить в следующем цикле.
- Ошибка Telegram: логировать и оставить событие в retry-очереди (опционально).
- Поврежденный state-файл: восстановиться из пустого состояния и зафиксировать warning.

## Формат уведомления (пример)
```text
ATAN: появилось топливо
АЗС: АЗС-101
Адрес: Автодорога А291...
Топливо: a95
Статус: FUEL_STATUS_IN_STOCK
```

## Соответствие статусам на сайте
- `FUEL_STATUS_IN_STOCK` -> "Все виды оплат" (зеленый)
- `FUEL_STATUS_LIMITED` -> "Талоны и топливные карты" (оранжевый)
- `FUEL_STATUS_OUT_OF_STOCK` / `FUEL_STATUS_UNAVAILABLE` -> "Временно не отпускается" (красный)
